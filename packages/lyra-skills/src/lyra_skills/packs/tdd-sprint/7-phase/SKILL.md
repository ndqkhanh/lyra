---
id: 7-phase
name: 7-Phase TDD Sprint
description: Disciplined RED → GREEN → REFACTOR cycle across a small feature slice.
---

1. **Localize.** Find the touch points.
2. **Plan.** Write down acceptance tests, expected files, forbidden files.
3. **RED.** Write the failing test; attach RED proof.
4. **GREEN.** Smallest diff to pass; no refactors.
5. **REFACTOR.** Clean up while tests stay green.
6. **Verify.** Objective + subjective verifier pass.
7. **Retro.** Capture what worked as a skill candidate.
