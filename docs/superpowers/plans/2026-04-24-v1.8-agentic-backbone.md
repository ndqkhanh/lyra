## Theme

The heaviest wave: turn every Wave-C surface into a real agent. The
slash commands are now load-bearing — `/red-proof` becomes the gate
`/tdd-gate` actually enforces; `/ultrareview` fans out to real
subagents in worktrees; `/btw` opens a sibling subagent with isolated
context; `/pair` streams thoughts in real time; `Ctrl+F` actually
kills running subagents. The CLI surface stays exactly where v1.7.5
put it; only the engine behind each slash gets replaced.

This wave is **mostly engine + lifecycle + permissions**. No new
slash commands (the matrix is at REPL parity already). No new themes,
no new keybinds. Difficulty per feature is high; the wave is large
because the engine has many independent moving parts.

## Invariants (from master roadmap)

1. **TDD-first** — every task writes the contract test before the
   implementation; the contract test imports from the new module
   before the module exists.
2. **No breaking changes** — every Wave-C slash continues to work
   identically; the engine wires *behind* the existing dispatchers,
   never changes their signatures.
3. **Evidence before assertion** — every `✓ shipped` parity-matrix
   flip names a code symbol AND a test file path.
4. **Optional deps** — `lyra[rl]`, `lyra[browser]`, `lyra[codeact]`,
   `lyra[vector]`, `lyra[mcp]` extras for the heavy adapters.
   Base install footprint must remain unchanged from v1.7.5.
5. **Graceful degradation** — every feature degrades to a friendly
   "feature requires `pip install lyra[<extra>]`" message when the
   extra is missing; no `ImportError` reaches the REPL.
6. **Local-first stays local-first** — no Wave-D feature pings the
   network unless the user already opted in via `/web` or
   `/mcp connect`.
7. **Plan-mode-default-on invariant** — the agent loop must refuse
   to write before the user explicitly enters `/mode build`.
8. **Worktree-only Edits** — every subagent that touches files runs
   in a `git worktree` rooted at `<repo>/.lyra/worktrees/<sub-id>`;
   no Edits land in the user's main checkout until the user accepts
   the diff via `/handoff` or `git apply`.

## Cross-wave dependencies

- `lyra_core.providers.preflight` (Wave B library): finally wired
  into the agent loop here (Task 14).
- `_TurnSnapshot` (Wave C, Task 1) gains a `tool_calls: list[ToolCall]`
  field in this wave so subagent fan-out replays cleanly.
- `RingBuffer` (Wave C, Task 4) becomes the substrate the
  `/blame`/`/trace` slashes use to surface real subagent traces.
- `BudgetCap` (Wave C, Task 5) gains live deduction inside the agent
  loop (Task 13) so `/budget set 5` actually halts spending at $5.

## Tasks (15 ordered units)

Each task lists:
- **Slashes / hooks** — the user-visible affordance.
- **Symbols** — the new code unit(s).
- **Test** — the contract-test file path + count.
- **Depends on** — Wave-D tasks that must land first.

  SubagentSpec, SubagentResult, SubagentStatus}` — orchestrator that

**Success criteria:**

- A user can write `~/.lyra/agents/researcher.yaml`, run
  `/spawn researcher: investigate X`, and the subagent runs to
  completion with worktree isolation.
- A user can `/spawn` 5 variants in parallel and pick the winner

  the agent loop respects: with `tdd_gate_enabled=True`, an Edit
  to `packages/foo/bar.py` is rejected unless the matching
  red-proof was recorded in the same turn.
