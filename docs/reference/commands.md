# Slash commands <span class="lyra-badge reference">reference</span>

Slash commands are intercepted by the CLI before the prompt reaches
the model. The unified registry at
[`lyra_cli/commands/registry.py`](https://github.com/lyra-contributors/lyra/tree/main/packages/lyra-cli/src/lyra_cli/commands/registry.py)
is the single source of truth — everything below is generated by hand
to mirror the runtime registry but the registry wins on conflict.

To see the current registry without launching a session:

```bash
python3 -c "from lyra_cli.commands.registry import COMMAND_REGISTRY; \
            print(*sorted((c.category, c.name) for c in COMMAND_REGISTRY), sep='\\n')"
```

Or in a session:

```
❯ /help
❯ /help <name>      # detailed help, args, examples
❯ /help all         # the full reference, paged
```

## Pipe substitutions

Inside a slash command, the following tokens are expanded against the
current selection / context (see
[`registry._PIPE_SUBS_RE`](https://github.com/lyra-contributors/lyra/tree/main/packages/lyra-cli/src/lyra_cli/commands/registry.py)):

| Token | Expands to |
|---|---|
| `%file` | The currently focused file |
| `%selection` | The currently selected text |
| `%line` | The current line number |
| `%project` | The repo root path |

A command that requires `%file` will not appear in tab-completion if
no file is focused.
