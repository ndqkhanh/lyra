    v2.1 "Claude-Code-Class" Rebuild — Implementation Plan¶

    For agentic workers: Execute task-by-task in this session. Each phase ends with a full-repo regression: pytest -q must show ≥1530 passed / ≤2 skipped / 0 failed. Steps use - [ ] syntax for tracking.

    Goal: Ship a production-grade Lyra CLI that lets users pick from any of 6 first-class providers (Anthropic, OpenAI, Gemini, DeepSeek, Qwen, Ollama), paste API keys interactively, and actually run agent tasks end-to-end. Remove mock from every user-facing surface. Adopt UX from claw-code and opencode.

    Architecture: In-place refactor with versioned module boundaries. Every phase keeps the test suite green. Default UX changes from day one — no env-var to opt in.

    Tech Stack: Python 3.11+, Typer (CLI), prompt_toolkit (REPL), Rich (TTY rendering), harness_core (agent loop), lyra_core (planner/auth/plan), pytest (TDD).

    Spec: docs/superpowers/specs/2026-04-26-production-rebuild-design.md

    No-git note: Per the user's directive, do not invoke git add / git commit between tasks. Verification gates are pytest runs.
