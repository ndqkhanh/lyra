    v1.8 "Agentic Backbone" — Wave-D Detailed Plan¶

    Created: 2026-04-24, immediately after Wave-C / v1.7.5 shipped. Status: READY FOR EXECUTION (start when v1.7.5 has been reviewed and the user gives the go-ahead). Predecessor: v1.7.5 "REPL Convergence" — shipped 2026-04-24 with +76 contract tests across 15 new files plus a +28 post-review safety patch, total 581 passed / 2 sandbox-skipped lyra-cli suite green (python3 -m pytest packages/lyra-cli). Master roadmap: 2026-04-24-full-parity-roadmap.md § Wave D. Successor: Wave E — v1.9 "Channels, Backends, Eval".

    Theme¶

    The heaviest wave: turn every Wave-C surface into a real agent. The slash commands are now load-bearing — /red-proof becomes the gate /tdd-gate actually enforces; /ultrareview fans out to real subagents in worktrees; /btw opens a sibling subagent with isolated context; /pair streams thoughts in real time; Ctrl+F actually kills running subagents. The CLI surface stays exactly where v1.7.5 put it; only the engine behind each slash gets replaced.

    This wave is mostly engine + lifecycle + permissions. No new slash commands (the matrix is at REPL parity already). No new themes, no new keybinds. Difficulty per feature is high; the wave is large because the engine has many independent moving parts.

    Invariants (from master roadmap)¶

    1. TDD-first — every task writes the contract test before the implementation; the contract test imports from the new module before the module exists.
    2. No breaking changes — every Wave-C slash continues to work identically; the engine wires behind the existing dispatchers, never changes their signatures.
    3. Evidence before assertion — every ✓ shipped parity-matrix flip names a code symbol AND a test file path.
    4. Optional deps — lyra[rl], lyra[browser], lyra[codeact], lyra[vector], lyra[mcp] extras for the heavy adapters. Base install footprint must remain unchanged from v1.7.5.
    5. Graceful degradation — every feature degrades to a friendly "feature requires pip install lyra[<extra>]" message when the extra is missing; no ImportError reaches the REPL.
    6. Local-first stays local-first — no Wave-D feature pings the network unless the user already opted in via /web or /mcp connect.
    7. Plan-mode-default-on invariant — the agent loop must refuse to write before the user explicitly enters /mode build.
    8. Worktree-only Edits — every subagent that touches files runs in a git worktree rooted at <repo>/.lyra/worktrees/<sub-id>; no Edits land in the user's main checkout until the user accepts the diff via /handoff or git apply.

    Cross-wave dependencies¶

    • lyra_core.providers.preflight (Wave B library): finally wired into the agent loop here (Task 14).
    • _TurnSnapshot (Wave C, Task 1) gains a tool_calls: list[ToolCall] field in this wave so subagent fan-out replays cleanly.
    • RingBuffer (Wave C, Task 4) becomes the substrate the /blame//trace slashes use to surface real subagent traces.
    • BudgetCap (Wave C, Task 5) gains live deduction inside the agent loop (Task 13) so /budget set 5 actually halts spending at $5.

    Tasks (15 ordered units)¶

    Each task lists: - Slashes / hooks — the user-visible affordance. - Symbols — the new code unit(s). - Test — the contract-test file path + count. - Depends on — Wave-D tasks that must land first.

    Task 1: Subagent runtime substrate (SubagentRunner)¶

    • Symbols: lyra_core.subagents.runner.{SubagentRunner, SubagentSpec, SubagentResult, SubagentStatus} — orchestrator that wraps an AgentLoop invocation in a worktree, captures stdio + HIR events, surfaces a typed result.
    • Test: lyra-core/tests/test_subagent_runner_contract.py — 6 tests (happy path, worktree isolation, cancellation, tool-output capture, error propagation, HIR scoping).
    • Depends on: —

    Task 2: SubagentRegistry (live process table)¶

    • Slashes: /agents + Ctrl+F (kill all) + /spawn upgraded.
    • Symbols: lyra_core.subagents.registry.{SubagentRegistry, spawn, kill, list_running, on_complete} — singleton process table with weak refs so the REPL never leaks.
    • Test: lyra-cli/tests/test_subagent_registry.py — 7 tests (register/spawn/list/kill/on_complete + status-line indicator + Ctrl+F empty case).
    • Depends on: Task 1.

    Task 3: User-defined subagent presets (~/.lyra/agents/<name>.yaml)¶

    • Symbols: lyra_core.subagents.presets.{load_user_presets, PresetSpec}; built-ins Explore, General, Plan, plus hermes-compat leaf / orchestrator aliases.
    • Test: lyra-cli/tests/test_subagent_presets.py — 6 tests (built-ins listed, user yaml loaded, alias resolution, missing file graceful, malformed file graceful, alias collision).
    • Depends on: Task 1.

    Task 4: DAG scheduler (fan-out + join)¶

    • Symbols: lyra_core.subagents.scheduler.{TaskGraph, Edge, schedule, run_to_completion} — topological executor with depends_on=[id, ...].
    • Test: lyra-core/tests/test_subagent_scheduler.py — 8 tests (linear chain, fan-out + join, diamond, cycle detection, cancellation cascades, partial-failure policy, deterministic order under shuffle, observability hooks).
    • Depends on: Tasks 1–2.

    Task 5: Variant runs (variants=N + LLM-judge selector)¶

    • Symbols: lyra_core.subagents.variants.{run_variants, VariantResult, judge_with_llm}; full PRM integration deferred to Wave F.
    • Test: lyra-core/tests/test_subagent_variants.py — 5 tests.
    • Depends on: Tasks 1, 4.

    Task 6: Permissions stack — destructive-cmd + secrets + injection guards¶

    • Symbols: lyra_core.permissions.{DestructiveCommandDetector, SecretsScanner, InjectionGuard, PermissionPolicy}; mode-based gating (strict | normal | yolo); regex packs + LLM-augmented classifiers; lyra[guards] extra for ML classifiers.
    • Test: lyra-core/tests/test_permissions_stack.py — 9 tests (rm -rf detected, secrets matched on AWS/Stripe/GitHub/GCP/Azure /Twilio, injection token detected, permission_mode honoured, allowlist via /less-permission-prompts).
    • Depends on: Wave-C permission_mode.

    Task 7: Tool-approval prompt (cached per session)¶

    • Slashes: every high-risk tool call shows [approve|reject|always].
    • Symbols: lyra_cli.interactive.approval.{ApprovalPolicy, ApprovalCache, request}; cache keyed by (tool_name, sha256(args)).
    • Test: lyra-cli/tests/test_tool_approval.py — 6 tests.
    • Depends on: Task 6.

    Task 8: ExecuteCode real (CodeAct sandbox via pyodide / firejail)¶

    • Symbols: lyra_core.tools.codeact.{ExecuteCode, PyodideRunner, FirejailRunner}; lyra[codeact] extra.
    • Test: lyra-core/tests/test_tool_codeact.py — 6 tests (stdout / stderr / return / timeout / sandbox-escape rejected / extra-missing graceful).
    • Depends on: Tasks 1, 6.

    Task 9: Browser real (playwright)¶

    • Symbols: lyra_core.tools.browser.{Browser, navigate, screenshot, click, type_text, evaluate}; lyra[browser] extra.
    • Test: lyra-core/tests/test_tool_browser.py — 5 tests (navigate→DOM, screenshot bytes, click no-such-selector graceful, evaluate returns JSON, extra-missing graceful).
    • Depends on: Tasks 1, 6, 7.

    Task 10: Custom user tools (~/.lyra/tools/<name>.py + @tool)¶

    • Symbols: lyra_core.tools.user.{ToolRegistry, register, invoke, _import_user_tools}; lazy import on first invocation; multiprocessing.Process worker for isolation.
    • Test: lyra-core/tests/test_user_tools.py — 6 tests.
    • Depends on: Tasks 1, 6.

    Task 11: Hooks lifecycle (SessionStart … TurnRejected)¶

    • Symbols: lyra_core.hooks.{HookSpec, HookRegistry, fire}; events SessionStart, SessionEnd, UserPromptSubmit, PermissionRequest, SubagentStart, SubagentStop, FileChanged (via watchdog), WorktreeCreate, ErrorThrown, TurnRejected.
    • Test: lyra-core/tests/test_hooks_lifecycle.py — 11 tests (one per event + plugin-handler ordering).
    • Depends on: Wave-B PluginRuntime.

    Task 12: MCP full (client + server + trust banner)¶

    • Slashes: /mcp connect, /mcp disconnect, /mcp list, /mcp serve.
    • Symbols: lyra_cli.mcp.{McpClient, McpServer, TrustBanner, ManifestSigner}; lyra[mcp] extra.
    • Test: lyra-cli/tests/test_slash_mcp.py — 8 tests (client connects to loopback, server accepts client, trust banner shown on first use, signed manifest verified, unsigned manifest rejected, disconnect graceful, list reflects state, extra-missing graceful).
    • Depends on: Tasks 6–7.

    Task 13: BudgetCap live deduction (Wave-C carry-over)¶

    • Symbols: integrate BudgetCap.enforce into the agent loop; refuse to start a turn when the next-turn estimate would push spend over EXCEEDED; emit BudgetAlertEvent.
    • Test: lyra-core/tests/test_agent_loop_budget.py — 5 tests.
    • Depends on: Wave-C BudgetCap.

    Task 14: Preflight wiring (Wave-B carry-over)¶

    • Symbols: wire lyra_core.providers.preflight into lyra_cli.providers.openai_compatible.{OpenAICompatibleLLM, AnthropicLLM, GeminiLLM, BedrockLLM, VertexLLM, CopilotLLM}; configurable ~/.lyra/settings.json → preflight: { mode: "warn"|"raise"|"off" }.
    • Test: lyra-cli/tests/test_provider_preflight_wired.py — 8 tests covering on/off/warn modes, stream-token estimation, tool-payload accounting, /compact suggestion banner, override via env.
    • Depends on: Wave-B preflight.

    Task 15: Live-streaming /pair + parity matrix + CHANGELOG + Wave-E plan¶

    • Slashes: /pair on upgrades to a real subscriber on the RingBuffer event stream; thoughts render incrementally.
    • Symbols: lyra_cli.interactive.pair.{PairStreamer, render}.
    • Test: lyra-cli/tests/test_slash_pair_streaming.py — 4 tests.
    • Docs:
    • Update docs/feature-parity.md: flip every Wave-D cell to ✓ shipped (v1.8); add §5e "v1.8 'Agentic Backbone' delta" with a row-per-feature table.
    • Append to CHANGELOG.md.
    • Write the detailed Wave-E plan (2026-04-24-v1.9-channels-backends-eval.md).

    Test target: ~95 new RED/GREEN contract tests.

    Success criteria:

    • A user can write ~/.lyra/agents/researcher.yaml, run /spawn researcher: investigate X, and the subagent runs to completion with worktree isolation.
    • A user can /spawn 5 variants in parallel and pick the winner via the LLM judge.
    • A user can install a third-party MCP server and the trust banner appears on first use.
    • lyra run --tool browser_navigate "https://…" opens a headless Chromium and returns the rendered DOM.
    • /budget set 5 halts the agent at $5 of cumulative spend (HIR emits BudgetAlertEvent and the UI surfaces a banner).
    • /red-proof packages/foo/tests/test_bar.py is now the gate the agent loop respects: with tdd_gate_enabled=True, an Edit to packages/foo/bar.py is rejected unless the matching red-proof was recorded in the same turn.

    Definition of done¶

    • All 15 tasks landed with both RED + GREEN tests committed.
    • Full lyra-cli + lyra-core suite green (python3 -m pytest packages/lyra-cli packages/lyra-core).
    • docs/feature-parity.md §1.4 (subagents) + §1.5 (memory) + §1.6 (permissions) + §1.7 (hooks) all flipped to ✓ shipped (v1.8).
    • CHANGELOG.md v1.8 section authored with the same shape as v1.7.5.
    • 2026-04-24-v1.9-channels-backends-eval.md exists with at least task-level granularity.
    • No new dependencies added to base install — every adapter behind a lyra[<extra>] extra.
