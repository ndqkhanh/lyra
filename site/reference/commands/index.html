    Slash commands reference¶

    Slash commands are intercepted by the CLI before the prompt reaches the model. The unified registry at lyra_cli/commands/registry.py is the single source of truth — everything below is generated by hand to mirror the runtime registry but the registry wins on conflict.

    To see the current registry without launching a session:

    python3 -c "from lyra_cli.commands.registry import COMMAND_REGISTRY; \
            print(*sorted((c.category, c.name) for c in COMMAND_REGISTRY), sep='\\n')"

    Or in a session:

    ❯ /help
❯ /help <name>      # detailed help, args, examples
❯ /help all         # the full reference, paged

    Session¶

    Command Aliases Args Description
    /help ?, h [name|all] List commands or detail one
    /status Cost, context fill, model, mode, tool calls
    /cost Cost only, broken down by model
    /clear Clear screen (transcript stays)
    /save Persist transcript + state right now
    /resume <id> Resume a previous session
    /exit /quit, q End the session

    Plan · build · run¶

    Command Aliases Args Description
    /mode <name> Switch mode: edit_automatically/ask_before_edits/plan_mode/auto_mode
    /mode show Dump the current system prompt
    /plan One-turn plan_mode
    /build One-turn edit_automatically
    /approve Approve current plan, switch to edit_automatically
    /reject Reject current plan, stay in plan_mode
    /run <task> Single-shot subagent task
    /diff Show diff produced this session
    /undo Revert last edit (uses git stash)

    Tools · agents¶

    Command Aliases Args Description
    /tools List enabled tools
    /tool <name> <args…> Call a tool directly, no model
    /agents List subagents in this session
    /spawn <task> Spawn a parallel subagent in a worktree
    /cycle Advance TDD phase (red→green→refactor→red)

    Observability¶

    Command Aliases Args Description
    /trace Open the trace viewer for this session
    /hir Show the HIR span tree
    /logs Tail this session's logs
    /why <id> Explain why a particular tool call ran
    /review [name] Run the review pass; optionally on one hook

    Config · theme¶

    Command Aliases Args Description
    /config [get|set|edit] Inspect or edit ~/.lyra/config.toml
    /model [name|fast=…|smart=…] Switch model slot
    /theme [name] Switch the colour theme

    Skill¶

    Command Aliases Args Description
    /skill <sub> Subcommands: list, add, remove, smoke, review, curator-report
    /skills Shorthand for /skill list

    MCP¶

    Command Aliases Args Description
    /mcp [sub] Subcommands: list, add, remove, save

    Plugin¶

    Command Aliases Args Description
    /plugin <sub> Subcommands: list, enable, disable

    TDD¶

    Command Aliases Args Description
    /tdd-gate <on|off|bypass-once> Enable/disable the TDD gate
    /red Force-enter red permission mode
    /green Force-enter green permission mode
    /refactor Force-enter refactor permission mode

    HUD¶

    Command Aliases Args Description
    /hud <sub> Subcommands: preset, presets, inline
    /hud preset <name> Switch HUD preset (minimal / compact / full / inline)

    Meta¶

    Command Aliases Args Description
    /version Show Lyra version
    /doctor Run the same checks as lyra doctor
    /feedback <text> Append a feedback note to the local feedback log

    Pipe substitutions¶

    Inside a slash command, the following tokens are expanded against the current selection / context (see registry._PIPE_SUBS_RE):

    Token Expands to
    %file The currently focused file
    %selection The currently selected text
    %line The current line number
    %project The repo root path

    A command that requires %file will not appear in tab-completion if no file is focused.
